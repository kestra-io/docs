This guide covers what you need to change when upgrading from **Kestra 1.3** to **2.0.0**.

:::alert{type="warning"}
**Start from Kestra 1.3.** This guide assumes you are already running the latest **1.3.x** release. If you are on an older version, complete the required metadata migrations before upgrading to 2.0.0:

- [KV Store and Secrets metadata migration](../v1.1.0/kv-secrets-metadata-migration) (introduced in 1.1)
- [Namespace Files metadata migration](../v1.2.0/namespace-file-migration) (introduced in 1.2)

If you are upgrading directly from 1.0, the [LTS migration guide (1.0 → 1.3)](../v1.3.0/lts-migration) consolidates all required steps in one pass.
:::

## What changed in 2.0.0

Kestra 2.0.0 removes several task properties and trigger condition types that were deprecated throughout the 1.x series but continued to work as aliases. If you completed the 1.3 migration, the changes in this guide are all that remain before upgrading.
